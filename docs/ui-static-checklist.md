# UI 분리 후 정적 점검 체크리스트

## 1) 네비게이션 전환
- [ ] 사이드바 워크스페이스 메뉴(`대화 연습/시나리오/복습/통계`) 클릭 시 `view`가 정상 전환되는가.
- [ ] 코치 도구 메뉴 클릭 시 `practice` 화면으로 이동하면서 대응 패널 탭이 열리는가.
- [ ] `오늘 추천` 클릭 시 추천 시나리오가 선택되고 시나리오 선택 패널이 열리는가.

## 2) 시나리오 선택 패널(showScenarioPicker)
- [ ] 대화 화면 상단 목록 버튼으로 시나리오 선택 패널을 토글할 수 있는가.
- [ ] 시나리오 선택 시 기존 세션 컨텍스트가 초기화되고 새 상태 안내 문구가 노출되는가.
- [ ] 라이브러리에서 `이 시나리오로 연습 시작` 동작이 연습 화면 전환 + 패널 닫힘을 유지하는가.

## 3) 설정 모달(showSettings)
- [ ] 사이드바 하단 설정 버튼과 TopBar 설정 버튼이 모두 동일한 설정 모달을 여는가.
- [ ] 오버레이 클릭, 닫기 아이콘 클릭 시 설정 모달이 닫히는가.
- [ ] API 키 미입력 상태에서 메시지 전송 시 설정 모달이 자동 오픈되는가.

## 4) 코치 패널(PracticePanelTab)
- [ ] `guide/challenge/suggestions/analysis/recap` 탭 전환이 기존과 동일하게 렌더링되는가.
- [ ] 같은 탭 재클릭 시 패널 닫힘 토글이 유지되는가.
- [ ] 챌린지 시작 시 기본 탭이 `challenge`로 이동하는가.

## 5) 구조 점검(변경 파일 범위 최소화)
- [ ] 레이아웃 컨테이너(`Sidebar`, `TopBar`, `PracticeShell`, `LibraryView`, `ReviewView`, `AnalyticsView`) 수정만으로 UI 도메인 단위 변경이 가능한가.
- [ ] 탭/패널 상태 변경은 `usePracticeUiContainer` 내부에서만 관리되는가.
- [ ] 스타일 수정 시 컴포넌트 경계(사이드바/탑바/프랙티스 셸) 단위로 영향 범위를 예측할 수 있는가.
